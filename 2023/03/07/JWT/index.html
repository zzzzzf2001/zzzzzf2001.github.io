JWT

JWT是什么

JSON Web Token ，通过数字签名的方式，以JSON对象为载体，在不同的服务终端之间安全的传输信息

JWT有什么用

JWT最常见的场景就是授权认证，一旦用户登录，后续每个请求都将包含JWT，系统在每次处理请求之前都要先进行JWT安全校验，痛过之后再进行处理

JWT的组成

JWT由三部分组成，用 . 来分隔

  • 第一部分 Header（eyJhbGcioiJIUzI1NiIsInR5cCI6IkpXVCJ9）

我们对一下的JSON数据进行base64编码变为以上的字符串,也就是我们编码后的TOKEN

  • 第二部分 payload 载荷 （.eyJ1c2VybmFtZS6IRvbsIsInJvbGUioijhZGlpbiIsInN1YiI6ImFkbwluLXR1c3QiLCJleHAiojE2MjMyMjM2NZUsImpOaSI6ImQ2MTIjzjcxLWI5zmUtNGMwNy04MZQwLTViowVizmMyNjExNyJ9）

载荷是存放有效信息的地方

  • 第三部分 Signatrue 签名

Signatrue由三部分构成第一个是编码之后的Header第二个是编码之后的payload第三部分是对于Header与payload的再次进行编码
