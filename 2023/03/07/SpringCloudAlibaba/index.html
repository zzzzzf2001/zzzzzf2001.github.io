Dubbo 2.7.7~2.7.8

为什么要用Dubbo

  • 服务治理框架
  • 服务的监控
  • 服务的注册和发现
  • 服务的通信
  • 服务的容错
  • 负载均衡

Dubbo的工作原理

SpringCloudAlibaba

组件：

  • Sentinel:把流量作为切入点，从流量控制、熔断降级、系统负载保护等多个维度保护服务的稳定性
  • Nacos：一个更容易构建云原生应用的动态服务发现、配置管理和服务管理平台
  • RocketMQ：一款开源的分布式消息系统，基于高可用分布式集群技术，提供低延时的，高可用的消息发布与订阅
  • Dubbo：Apache Dubbo是一款高性能的Java RPC框架
  • Seata：阿里巴巴的开源产品，一个易于使用的高性能微服务分布式解决方案
  • Alibaba Cloud ACM：一款在分布式架构环境中对应用配置进行集中管理和推送的应用配置中心产品
  • Alibaba Cloud OSS：阿里云对象存储服务（Object storage Service 简称OSS），是阿里云提供海量、安全、低成本、高可靠的云存储服务。可以在任何应用任何时间、任何地点存储和访问任意类型的数据
  • Alibaba Cloud schedulerX：阿里中间件团队开发的一款分布式任务调度产品，提供秒杀、精准、高可靠、高可用的定时（基于Cron表达式）任务调度服务
  • Alibaba Cloud SMS：覆盖全球的短信服务，友好、高效、智能的胡莲花通讯能力，帮助企业迅速搭建客户触达通道

Nacos

简介

Nacos是一个更易于构建云原生应用的动态服务发现、配置管理和服务管理平台，他是SpringCloud Alibaba组件之一，负责服务注册发现和服务配置，可以这样任务nacos = eureka+config

Nacos致力于帮助开发者发现、配置和管理微服务，Nacos提供了一组简单易用的特性集，帮助您快速实现动态发现、服务配置、服务元数据以及流量管理

上面介绍可以看出nacos的作用就是一个注册中心，用来管理注册上来的各个服务

负载均衡

什么是负载均衡?

通俗的将，负载均衡就是将负载（工作任务，访问请求）进行分摊到多个单元（服务器组件上进行）。根据负载均衡的发声位置不同，一般分为服务端负载均衡和客户端负载均衡

  • 服务端负载均衡是指发生在服务提供者一方，比如常见的nginx负载均衡
  • 而客户端负载均衡是指：发生在服务端请求一方，也就是在发送请求之前就已经选好了由哪个实例处理请求

那么我们将如何实现客户端的负载均衡呢？（看下一章）

Ribbon

什么是Ribbon？

Ribbon是SpringCloud的一个组件，他让我们使用一个注解就可以轻松的搞定服务端的负载均衡

在演示之前我们要对一个服务进行创建几个实例，修改虚拟机中的配置，-Dserver.port=8082 -Dserver.port=8083

利用Ribbon进行负载均衡处理

在OrderApplication（请求端启动类）中的 RestTemplate中添加注解@LoadBalanced（负载均衡）

在OrderController中直接去请求product模块服务，将之前的主机号和端口号省去变成了 服务名（service-product）

什么是feign
